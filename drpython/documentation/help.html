Вступ

Що таке DrPython?

drPython ? це просте і легке у використанні інтегроване середовище розробки (IDE) для мови програмування Python. Він розроблений для навчання та швидкого створення невеликих проєктів, що робить його популярним серед початківців та викладачів.

Як з'явився drPython?

  • drPython був створений як альтернатива більш складним IDE для Python, таким як PyCharm або Visual Studio Code.

  • Його розробили з акцентом на простоту та зручність, щоб навчати програмуванню на Python без зайвих складнощів.

  • drPython базується на Python та бібліотеці WxPython, що забезпечує крос-платформовість (працює на Windows, macOS, Linux).

Основні можливості drPython

  1. Простий інтерфейс:

    • Мінімалістичний дизайн, який не відволікає від написання коду.

    • Підтримка підсвічування синтаксису для Python.

  2. Підтримка інтерактивного режиму:

    • Вбудований інтерпретатор Python для швидкого тестування коду.

  3. Переглядач структури:

    • Інструмент для навігації між функціями, класами та методами у вашому коді.

  4. Автодоповнення коду:

    • Допомагає швидко писати код, пропонуючи можливі варіанти під час введення.

  5. Підсвічування помилок:

    • Визначає синтаксичні помилки та підсвічує їх у коді.

  6. Кросплатформність:

    • Працює у різних операційних системах: Windows, macOS, Linux.

  7. Підтримка доповнень (плагінів):

    • Можна розширювати функціональність за допомогою додаткових модулів.

  8. Легкість у використанні:

    • Ідеально підходить для початківців, які тільки починають вивчати Python.

Безкоштовність drPython

  • Безкоштовність полягає у тому, що drPython безкоштовний та відкритий для модифікацій. Це означає, що ви можете завантажити drPython безкоштовно, ділитися ним з ким і скільки завгодно, переглядайте джерельений код і навіть можете взяти фрагменти коду та додати їх у свою власну програму (за умови, що ви згадаєте авторів).
  • Ліцензія: drPython випущено під ліцензією GNU Public License.
     

Для кого підходить drPython?

  • Початківці: для тих, хто тільки починає вивчати Python.

  • Викладачі: для навчання програмуванню у школах чи університетах.

  • Швидка розробка: для створення невеликих скриптів або проектів.

Якщо  вам потрібен простий і легкий інструмент для навчання або швидкої роботи, drPython ? це гарний вибір!

Основні терміни

  • Cередовище програмування — програмний засіб, що має набір інструментів, які використовуються для створення, редагування, тестування та налагодження програмного коду.

  • Вікно редактора коду - вікно в якому відбувається створення та редагування програми. Можлива одночасна робота над декількома файлами у різних вкладках.

  • Консоль - вікно, у якому відбувається виконання програми (виведення та введення даних, виведення повідомлень про помилки).

  • Навігатор - вікно зі структурою програми, яке можна використати для швидкого переходу до різних її частин.

  • Додаткові панелі - панелі з вікнами, які можна використати для перегляду та редагування різних частин одного файлу або декількох файлів.

  • Вкладки - додаткові вікна для редагування програм у вікні редактора коду.

  • Панель меню - панель з меню у верхній частині вікна середовища програмування, яка надає доступ до більшості функцій.

  • Панель інструментів - панель з піктограмами-ярликами найбільш вживаніших функцій середовища програмування.

  • Закладки - назви файлів або тек, за якими їх можна швидко відкрити.

  • Вигулькне меню - меню, яке з'являється, коли ви клацаєте правою кнопкою миші у вікнах радектора коду(котекстне меню) або в Консолі.

  • Рядок стану - панель з інформацією про поточний стан середовища, показує [поточний рядок] [поточну позицію у рядку] [тип закінчення рядка] [режим вставлення/передруку] та статус програми/інтерпретатора Python.

  • Сполучення клавіш - комбінації клавіш, які можуть бути натиснуті одночасно для виклику функцій середовища програмування.

  • Налаштування - зміна параметрів середовища програмування відповідно до уподобань користувача, налаштування можуть бути збережені для подальшого використання.

  • Відступи - зміщення початку рядка коду відносно лівої межі, які визначають блоки вказівок у Python.

  • Індикатори пропусків - символи, які візуалізують наявність пропусків у тексті(символів SPACE) у вигляді точок.

  • Символи закінчення рядка - невідтворювані символи, які у файлах розділяють рядки тексту.

  • Розділювач тексту - коментар, який використовується для візуального розділення тексту з кодом програми.

  • Регулярні вирази — це спеціальні текстові шаблони, які використовуються для пошуку, зіставлення та маніпулювання рядками на основі певних правил.

Панель меню

Панель меню надає доступ до використовуваних у середовищі drPython інструментів для роботи з файлами, їх редагування, пошуку та заміни послідовностей символів, виконання коду, налаштування середовища та встановлення/вилучення доповнень.

 Файл

  • Новий

Створити новий файл для редагування. Насправді створюється нова вкладка у вікні редактора коду, файл буде створено про його збереженні.

  • Відкрити

Відкрити існуючий файл з кодом програми на Python. За замовчуванням буде використано теку drPythonProjects, користувач може обрати інше місце розміщення файлів.

  • Відкрити нещодавнiфайли...

Відкрити файли зі списку останніх збережених файлів.

  • Закладки

Відкрити файли або теки  зі списку закладок. Закладки додаються/видаляються у Налаштуваннях.

  • Перезавантажити файл

Перезавантажити файл, у випадку, якщо потрібно повернутись до раніше збереженої версії коду програми.

  • Відновити з резервної копії

Відновити файл з резервної копії, допоможе повернутись до попередньої версії коду програми навіть у випадку, якщо файл було вже перезаписано.

  • Відкрити імпортований модуль...

Відкрити код імпортованого модуля. Використовуйте для імпортованих модулів, створених користувачем.

  • Закрити

Закрити файл, якщо файл було модифіковано з часу останнього збереження, тоді буде запропоновано зберегти його.

  • Спорожнити список нещодавніх файлів

Спорожнити список останніх використаних файлів.

  • Зберегти

Зберегти файл з кодом програми, якщо файл не було раніше збережено, тоді буде запропоновано обрати його назву. До назв файлів автоматично додається розширення .py . За замовчуванням файли зберігаються у теці drPythonProjects, користувач може обрати інше місце розміщення файлів.

  • Зберегти як...

Зберегти файл, обравши його назву. Доцільно, якщо користувач хоче зберегти копію коду під іншою назвою. До назв файлів автоматично додається розширення .py . За замовчуванням файли зберігаються у теці drPythonProjects, користувач може обрати інше місце розміщення файлів.

  • Зберегти яккопію

Зберегти копію існуючого файлу, який було відкрито для редагування.

  • Зберегти вмicтконсолі до файлу...

Збереження текстового вмісту панелі консолі до текстового файлу.

  • Налаштування друку...

Налаштування друку з використанням можливостей системної друкарки(принтера)

  • Друк файлу...

Друк файлу. який містить код програми з поточної вкладки редактора коду.

  • Друк вмісту консолі

Друк текстового вмісту панелі консолі.

  • Вихід

Вихід з програми за запитом. Буде запропоновано зберегти модифіковані файли вкладок редактора коду.

Не можна змішувати у відступах табуляції та пропуски. Ви можете:

- збільшити відступ

- зменшити відступ

- визначити тип відступів (пропусками або табуляціями)

- встановити відступи табуляціями

- встановити відступи пропусками

Можна налаштувати закінчення рядків:

- закінчення рядків як Unix/Linux ('\n')

- закінчення рядків як DOS/Windows ('\r\n')

- закінчення рядків як Мас OS ('\r')

За замовчуванням встановлено закінчення рядків '\n'

Структура - об'єкт, який містить назви використаних у коді модулів, класів та функцій. Структура відображається на панелі ліворуч основного вікна. Клацок по відповідній назві викликає перехід до відповідного елемента у вікні редактора коду. Надається можливість пошуку об'єкта структури за його назвою.

  • Перейти за структурою...

Перехід за елементами структури, які подані в окремому вікні, без виклику бічної панелі.

  • Показати/сховати індикатори пpoпycків

Візуалізація символів "пропуск"(space) у тексті. Може бути використана для контролю відступів.

  • Показати/сховати консоль

Консоль - додаткова панель, яка знаходиться нижче вікна з кодом програми. Призначення консолі - відтворювати результати роботи програми, які передаються через стандартний канал  виведення та отримувати дані від користувача через стандартний канал введення. Також у консоль виводяться сповіщення про помилки, які виникають під час роботи програми.

Закладки

  1. Закладки та теки із закладками:

    • Закладки призначені для швидкого доступу до потрібних файлів та тек з файлами без потреби пошуку у файлових діалогах. Принцип роботи закладок схожий на використання списку останніх файлів, при цьому користувач може власноруч додавати та видаляти закладки, які можуть бути закладкими файлів та файлових тек. Закладки можна організовувати, зберігаючих їх у теках закладок.

    • Закладка -  елемент, який може бути або файлом, або файловою текою.

    • Папка із закладками - контейнер, який може містити інші закладки або теки із закладками, утворюючи ієрархічну структуру.

  2. Функціональність Закладок:

    • Якщо закладка є файлом, її вибір із меню призводить до відкриття цього файлу.

    • Якщо закладка є текою, її вибір відкриває діалогове вікно для навігації у ній.

  3. Додавання закладок:

    • Додавати нові закладки або теки із закладками можна лише всередину іншої теки із закладками. Це дозволяє організовувати елементи у структуровану систему.

  4. Переміщення закладок:

    • Для переміщення закладки або теки із закладками використовується метод "перетягування" (drag-and-drop).

    • Елемент можна перемістити після конкретного елемента в папці або зробити його першим елементом у новій теці.

Перевірка синтаксису

Перевірка синтаксису - перевірка правильності коду у вікні редактора. Для перевірки код має бути попередньо збережений у файлі. У середовищі розробки drPython перевірка синтаксису здійснюється у різний спосіб:

  1. Підсвітка синтаксису: drPython автоматично підсвічує ключові слова, коментарі, рядки та інші елементи коду, що допомагає візуально визначити помилки.

  2. Перевірка під час запуску: Якщо у коді є синтаксичні помилки, drPython покаже повідомлення про помилку під час спроби запустити код. Це дозволяє швидко виправити проблеми.

  3. Автоматична перевірка: Ви можете атоматично перевірити синтаксис, вибравши відповідну опцію в меню. Якщо код містить помилки, вони будуть вказані у вікні виводу. Зверніть увагу, що це перевіряє лише синтаксис поточного файлу. 

  4. Інтерактивна консоль: drPython має вбудовану інтерактивну консоль Python, яка дозволяє виконувати код по рядках і відразу бачити результат, що допомагає швидко знаходити помилки.

 

Перевірка синтаксису через розділ меню "Перевірити синтаксис"  працює на основі вбудованих можливостей інтерпретатора Python. Це відбувається наступним чином:

  1. Зчитування коду:

    • IDE зчитує весь код, який знаходиться у відкритій вкладці вікна редактора.

  2. Передача коду інтерпретатору Python:

    • drPython передає цей код інтерпретатору Python для перевірки синтаксису. Це робиться через вбудований механізм, який викликає функцію compile() з Python.

  3. Компіляція коду:

    • Інтерпретатор Python намагається скомпілювати код. Якщо код містить синтаксичні помилки, інтерпретатор повертає виняток (exception), наприклад, SyntaxError.

  4. Обробка результату:

    • Якщо код не містить помилок, drPython показує повідомлення про те, що синтаксис правильний.

    • Якщо є помилки, drPython відображає повідомлення про помилку, вказуючи тип помилки, рядок та позицію, де виникла проблема.

  5. Відображення помилок:

    • IDE підсвічує рядок з помилкою у редакторі коду, щоб користувач міг швидко її виправити.

Пошук і заміна

У drPython функція заміни тексту дозволяє швидко знаходити та змінювати певні фрагменти коду або тексту.

  1. Відкриття вікна пошуку:

    • Це можна зробити через меню Пошук > Пошук , через панель інструментів або за допомогою сполучень клавіш (Ctrl + F).

  2. Відкриття вікна заміни:

    • Це можна зробити через меню Пошук > Заміна або за допомогою сполучень клавіш (Ctrl + R).

  3. Введення тексту для пошуку:

    • У поле Шукати введіть текст або фрагмент коду, який ви хочете знайти.

  4. Введення тексту для заміни:

    • У поле Замінити на введіть новий текст, на який потрібно замінити знайдений фрагмент.

  5. Опції заміни:

    • У виділеному: замінює всі знайдені збіги у виділеній області.

    • Заміна за запитом: дозволяє вручну перевіряти та підтверджувати кожну заміну, щоб автоматичн замінити всі входження шуканого, зніміть позначку з цієї опції. 

    • Враховувати регістр: якщо ця опція активована, пошук буде враховувати регістр літер.

    • Регулярний вираз: дозволяє використовувати складні шаблони для пошуку.

  6. Контекстне меню:

    • Клацання правою кнопкою миші на текстовому полі призведе до появи меню, за допомогою якого можна додати керуючі символи, використати вміст буферу обміну або видалити текст.

    • Натиснувши трикутник у полі введення, ви можете отримати доступ до списку історії пошуку/заміни.

Автодоповнення коду

Функція Автодоповнення у drPython - це інструмент, який допомагає розробникам швидше писати код, автоматично пропонуючи можливі варіанти завершення команд, функцій, змінних, модулів або інших елементів коду під час введення. Ось детальний опис того, як ця функція працює:

Як працює Autocomplete у drPython:

  1. Аналіз коду:

    • Коли ви починаєте вводити код, drPython аналізує ваш поточний файл, визначаючи доступні ключові слова, функції, класи, змінні та інші елементи.

  2. Підказки під час введення:

    • Після натиску клавіші MENU drPython автоматично відображає список можливих варіантів для автодоповнення.

    • Наприклад, якщо ви введете pri, drPython запропонує print як один із варіантів.

    • Автодоповнення чутливе до регістру символів.

  3. Швидкий вибір варіантів:

    • Ви можете вибрати потрібний варіант із списку автодоповнення за допомогою миші або клавіші Tab для автоматичного завершення.

    • Переміщуйтеся по списку за допомогою клавіш Up/Down, щоб рухатися швидше - PageUp/PageDown, або Home/End щоб перейти до початку та кінця списку.
    • Натискання клавіші Enter або Tab завершить автодоповнення.

    • Залежно від контексту автодоповнення зупиняється на таких символах: пропуску, (, ), <, >, [, ], {, }

Навігатор структури

Навігатор структури (Source Browser) у drPython - це інструмент, який дозволяє швидко переглядати та навігувати між різними частинами вашого коду, зокрема функціями, класами та методами. Він особливо корисний у великих проєктах, де потрібно швидко знаходити певні елементи коду.

Як користуватися Переглядачем структури:

  1. Відкрийте панель Навігатор:

    • У головному меню drPython виберіть Вигляд -> Показати/сховати Навігатор структури.

    • Або скористайтеся гарячою клавішею F8.

  2. Інтерфейс Навігатора структури:

    • У вікні панелі Навігатора ви побачите список усіх функцій, класів та методів у вашому файлі.

    • Елементи розміщені у порядку їхнього слідування у програмі, з відступами для вкладених конструкці (наприклад, функції, класів тощо) і можуть бути розгорнуті для перегляду вкладених методів.

  3. Навігація:

    • Клацніть на будь-який елемент у списку (наприклад, ім'я функції чи класу), і drPython автоматично перемістить курсор до відповідного місця у вашому коді.

  4. Фільтрація:

    • Якщо у вас багато елементів, ви можете скористатися полем пошуку у нижній частині панелі Структура, щоб швидко знайти потрібний елемент.

  5. Оновлення:

    • Якщо ви внесли зміни до коду, натисніть кнопку Оновити, щоб оновити список елементів.

Сполучення клавіш

Існує досить велика кількість сполучень клавіш для швидкого виклику різних функцій у drPython. Разом з цим, ви зможете призначити власні комбінації клавіш для зручного використання програми.

1. Як відкрити меню налаштувань сполучень клавіш:

  • У головному меню drPython виберіть "Налаштування".
  • У розділі меню "Налаштування знайдіть пункт "Налаштувати сполучення клавіш" та натисніть на нього.

2. Ігнорування модифікаторних клавіш

Якщо ви працюєте на Linux або іншій системі, де деякі модифікаторні клавіші (наприклад, "Meta" або "Num Lock") завжди активні, скористайтеся функцією "Ігнорувати клавіші" (Ignore Keys).

У вікні налаштувань знайдіть поле "Ignore Keys" та введіть туди модифікаторні клавіші, які потрібно ігнорувати (наприклад, "Meta").

Це допоможе уникнути конфліктів із системними комбінаціями клавіш.

3. Призначення сполучень клавіш для DrScript

Якщо ви використовуєте DrScript (скрипти для drPython), ви можете налаштувати сполучення клавіш для їх швидкого виклику.

У вікні налаштувань сполучень клавіш знайдіть пункти, пов’язані з DrScript.

Якщо сполучення клавіш для DrScript не встановлені або файл із налаштуваннями не синхронізований, drPython автоматично завантажить правильні скрипти без сполучень клавіш за замовчуванням.

4. Створення нового сполучення клавіш

У вікні налаштувань сполучень клавіш знайдіть функцію, для якої потрібно призначити сполучення клавіш. Назви функцій наведено англійською мовою для сумісності з англомовною версією програми.

Натисніть кнопку "Отримати клавішу" (Get Key).

Натисніть бажану комбінацію клавіш на клавіатурі (наприклад, Ctrl + S або Alt + F).

Не використовуйте комбінацій зарезервованих операційною системою, таких як Alt+Shift або Ctrl+Shift.

Програма відобразить виявлені модифікатори (наприклад, Ctrl, Alt) та код клавіші.

Натисніть "Закрити" (Close), щоб зберегти комбінацію.

5. Збереження та застосування змін

Після налаштування сполучення клавіш натисніть "Оновити" (Update), щоб негайно застосувати зміни.

Щоб зберегти зміни для майбутніх сеансів роботи з drPython, натисніть "Зберегти" (Save).

6. Особливості роботи з деякими клавішами

Tab та Enter використовуються для роботи з відступами та командним рядком. Налаштування цих клавіш у сполученнях може призвести до неправильної роботи drPython.

На деяких платформах деякі клавіші можуть не реєструватися, тому їх неможливо використовувати для сполучень.

Якщо виникає проблема з комбінацією Shift + Backspace (вона може вставляти символ Backspace), призначте цю комбінацію для функції "Видалити назад, але не весь рядок" (Delete Back Not Line). Якщо потрібно вставити символ Backspace, просто видаліть це сполучення.

7. Перевірка роботи сполучень клавіш

Якщо сполучення клавіш не працюють, скористайтеся кнопкою "Отримати клавішу" (Get Key), щоб перевірити, чи правильно визначено комбінацію.

Якщо модифікаторна клавіша (наприклад, "Meta" на Linux) постійно активна, додайте її до списку ігнорованих клавіш у розділі "Ignore Keys".

8. Важливі зауваження

Сполучення клавіш працюють коли фокус введення знаходиться у вікні редактора коду. Ви можете просто клацнути вказівником миші по його вікні, щоб активізувати його.

У деяких операційних системах комбінації клавіш можуть працювати лише в розкладці з латинськими(англійськими) літерами.

Сполучення клавіш для DrScript зберігаються окремо від основних сполучень drPython.

Якщо виникають проблеми з роботою сполучень, перевірте, чи не конфліктують вони із системними комбінаціями клавіш або іншими налаштуваннями.

Використання додаткових панелей

Перегляд на панелі дозволяє переглядати будь-який відкритий документ (включаючи поточний один) на будь-якій додатковій панелі.

У drPython додаткові панелі дозволяють розділити вікно редактора для одночасної роботи з різними частинами одного файлу або для перегляду вмісту інших файлів. 

Як створити додаткову панель:

  1. Відкрийте drPython.

  2. Перейдіть у меню "Перегляд" .

  3. Виберіть опцію "Показати у панелі". Зазвичай є такі варіанти:

    • Показати у верхній панелі: вікно розділиться на верхню та нижню частини.

    • Показати у лівій або правій панелі: вікно розділиться на ліву та праву частини.

  4. У відповідь на запит оберіть вміст якого файлу(вкладки) буде відображено у додатковій панелі.

  5. У додатковій панелі ви побачите вміст обраного файлу. Ви можете прокручувати їх незалежно, щоб працювати з різними частинами коду.

Як відкрити інший файл у додатковій панелі:

  1. У меню "Файл" (File) та виберіть "Відкрити" (Open), щоб відкрити вкладку з вмістом файлу. 

  2. Створіть додаткову панель та відкрийте у ній потрібний файл, як описано вище.

  3. Файл відобразиться у додатковій панелі.

Як переміщуватися між панелями:

  • Ви можете клацати мишею у будь-яку панель, щоб активувати її.

  • Також можна використовувати гарячі клавіші (якщо вони підтримуються) для швидкого перемикання між панелями.

Як закрити додаткову панель:

  1. Натисніть кнопку "Закрити" (Close).

  2. Якщо було закрито вкладку редактора з вмістом додаткової панелі, тоді цю панель маєте закрити вручні

Використання:

  1. Порівняння частин коду: Ви можете розділити вікно, щоб одночасно переглядати різні частини одного файлу (наприклад, функції або класи).

  2. Робота з кількома файлами: Відкрийте два файли у різних панелях, щоб порівнювати або копіювати код між ними.

  3. Тестування та налагодження: Ви можете відкрити файл з кодом у одній панелі та інтерактивну консоль Python у іншій для швидкого тестування.

Розташування панелей:

  • Панелі можна розташовувати зверху,  ліворуч або праворуч від основного вікна, залежно від ваших уподобань.

  • drPython дозволяє перетягувати межі панелей мишею, щоб змінити їх розмір.

Запуск та зупинка програми Python

У середовищі програмування drPython запуск та зупинка програм виконуються наступним чином:

1. Запуск програми:
   - Натисніть кнопку Виконати (або F5 на клавіатурі), щоб виконати код.

2. Зупинка програми:
   - Якщо програма працює у нескінченному циклі(як у tkinter) або виконується тривалий час, натисніть кнопку Зупинити (або Ctrl+Q на клавіатурі), щоб зупинити виконання.

Якщо це не працює, можна просто закрити вікно програми користувача або перезапустити середовище.

Для передавання аргументів вашій програмі скористайтесь розділом меню Встановити аргументи.
